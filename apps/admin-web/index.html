HXYOS 工作台

前台只保留一个问答框和场景工作流。业务人员先完成动作，中台能力和后台治理在后面承接。

前台接待 前台怎么说：顾客问价格、效果、区别时，直接拿一句能说出口的话。 进入前台速查 对外发布 这句话能不能发：朋友圈、海报、短视频先做动作前预检。 检查对外表达 首店开业 开业还差什么：看首店当前主线、证据缺口和下一步动作。 进入开业推进 资料入库 资料变成知识：把文章、PDF、会议内容整理成后续能用的项目资料。 进入资料处理 员工训练 把标准说法变成可练、可纠偏、可复训的员工动作。 进入训练 H5 经营复盘 把问题、纠偏和结果沉淀到经营大脑，形成下一轮动作。 进入经营大脑

中台

知识引擎、检索应用、Skill、Agent、记忆

这里承接前台动作，把资料、标准答案、流程能力和组织记忆变成可调用对象。

知识引擎 资料、答案卡、标签、来源和有效范围的统一底座。 查看知识台 检索应用 不同场景用不同召回策略，不把所有问题丢给同一个 RAG。 查看应用 Skill 把合规检查、话术训练、资料编译沉淀成可测试能力。 查看技能 Agent 按意图选择工作流、工具和证据，不做无边界万能助手。 进入大脑 记忆 过程只作提醒，正式知识必须来自已核定资产。 查看记忆
后台

审核、权限、版本、运行记录、评测、监控

这些是管理动作，不放到前台干扰一线使用，但必须存在并可追溯。

审核 对外表达、标准答案和资料结论进入人工确认。 进入复核 权限 按角色控制查询、写入、执行和发布范围。 查看边界 版本 正式知识不原地覆盖，只能新版本替换旧版本。 查看版本 运行记录 资料处理、Loop、工作流调用都要留下可追溯记录。 查看记录 评测 用 benchmark 证明答案、引用和风险拦截是否达标。 查看评测 监控 关注失败任务、成本、接口状态和质量漂移。 查看状态
