# 荷小悦项目菜单产品与 UI 设计

## 目标

将 `wellness-order.html` 从写死的点单页升级为三端闭环：

- 用户前端：选择服务、查看详情、选择清泡调补养方案、下单。
- 管理后台：维护分类、项目、价格、图片、标签、服务步骤、技师说明和清泡调补养绑定。
- 技师前端：查看待服务订单、识别顾客选择、按步骤履约并更新状态。

## 设计读法

这是门店 H5 服务菜单，不是普通电商商城。视觉应保持白底、浅翠绿、干净、疗愈、可信任，同时要让门店现场操作足够快。

## 信息架构

### 用户端

- 顶部：选择服务、品牌副标题、搜索入口。
- 左侧：分类导航。
- 右侧：分组项目列表。
- 项目卡：图片、名称、标签、时长、现价、划线原价、加号。
- 详情页：大图、项目卖点、清泡调补养绑定、必选项、手法偏好、特色加购、服务说明、购买栏。
- 订单确认：手机号、座位、备注、已选项目、总价。

### 管理后台

- 密码进入。
- 分类列表与项目列表。
- 项目编辑表单：名称、分类、图片、价格、原价、时长、标签、功效、服务步骤、技师说明、推荐话术。
- 清泡调补养编辑：`清`、`泡`、`调`、`补`、`养` 分别维护。
- JSON 原始编辑区：允许修改任意前端项目信息。
- 发布后写入 catalog API。

### 技师端

- 密码进入。
- 订单按状态筛选：待确认、服务中、已完成。
- 订单卡展示：门店、座位、手机号、项目、备注、总价。
- 服务详情展示：清泡调补养选择、服务步骤、技师说明、推荐话术。
- 操作：确认服务、完成服务。

## 数据模型

Catalog 使用 JSONB 存储，保持前端可扩展：

```json
{
  "brand": {},
  "categories": [],
  "projects": []
}
```

项目核心字段：

```text
id, categoryId, status, featured, name, subtitle, coverImage,
price, originalPrice, durationMinutes, tags, benefits,
serviceSteps, technicianNotes, recommendScript,
qingPaoTiaoBuYang, requiredOptions, optionalOptions, addOns
```

## 第一版实现边界

- 不接入若依后台。
- 不引入复杂权限系统。
- 管理后台先用 H5 密码门禁。
- Catalog 通过 htops wellness API 持久化。
- 用户端、技师端、管理端均先作为独立 HTML 页面交付。
