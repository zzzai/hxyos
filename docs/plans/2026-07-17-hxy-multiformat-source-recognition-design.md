# HXY 多格式资料识别与解析路由设计

## 产品决策

HXYOS 不使用 MinerU 重新“理解”全部原始资料。MinerU 只是多格式工具链中的一个高保真解析器，主要处理扫描、版式复杂、图片或表格较多的文档。

正确链路是：

```text
原始资料
-> 源文件治理与去重
-> 轻量结构预检
-> 解析器路由
-> 解析质量闸
-> 必要时备用解析器
-> 结构化资料包
-> 模型按业务任务理解
-> 少量核心知识治理
```

解析器只回答“文件内容是否被正确读出”。模型理解层回答“资料在荷小悦当前阶段意味着什么”。治理层回答“哪些结论可以成为荷小悦正式口径”。三者不得混用。

## 第一性原则

1. 原始文件不可覆盖，所有解析结果都是可重建的衍生产物。
2. 先判断文件和结构，再选择工具，不按扩展名批量调用最重工具。
3. 解析质量不等于来源可信度，也不等于内容正确性。
4. 外部资料和 AI 衍生资料可以辅助判断，不能自动成为荷小悦事实。
5. 自动流程应在异常处停住，不为每个文件、段落或 claim 制造人工审核任务。
6. 工具失败可自动回退；所有工具都失败时才进入异常处理。
7. 模型只接收通过解析质量闸、带来源元数据和证据定位的资料包。

## 分层职责

### 1. Source Registry

对文件做只读盘点、哈希去重和源文件级治理分类：内部项目、内部记录、外部资料、AI 衍生、解析产物、工具文件。该层不读取业务含义，不批准知识。

### 2. Document Preflight

使用低成本、确定性的方式抽样结构：

- PDF：页数、代表页文字密度、代表页图片信号；
- DOCX/PPTX/XLSX：OpenXML 页面或工作表数量、媒体和表格信号；
- 文本文件：编码、长度和基本结构；
- 图片：后续增加尺寸、方向、OCR 可读性和场景类型检测；
- 链接、音频、视频：后续进入独立采集适配器，不伪装成普通文件。

预检有严格的页数和字节预算，不调用大模型，不做全量 OCR。

### 3. Parser Router

当前工具矩阵：

| 资料类型 | 首选 | 备用 | 原因 |
|---|---|---|---|
| Markdown/TXT | HXY 文本编译器 | 无 | 已是可读文本 |
| HTML/EPUB/CSV/JSON | MarkItDown | 异常处理 | 轻量结构化转换 |
| 普通文本型 PDF | MarkItDown | MinerU | 先低成本读取，质量不足再高保真解析 |
| 扫描或图片主导 PDF | MinerU | MarkItDown | 需要 OCR 和版面恢复 |
| 普通 DOCX | MarkItDown | MinerU | 轻量提取优先 |
| 媒体、表格或复杂布局 DOCX | MinerU | MarkItDown | 需要更完整结构 |
| PPTX | MinerU 或 MarkItDown | 另一解析器 | 根据媒体和布局信号选择 |
| XLSX | MarkItDown 或 MinerU | 另一解析器 | 根据表格复杂度选择 |
| XLS | MarkItDown | 异常处理 | 当前 MinerU 不支持旧版 XLS |
| DOC/PPT | 异常处理 | 无 | 当前工具均无可靠直接支持，需先配置安全转换器 |
| 图片 | OCR/视觉适配器 | 异常处理 | 需区分文档扫描、截图、门店照片和设计图；不直接路由 MinerU |

本机已验证的版本边界：MinerU `3.4.2` 支持 PDF、图片、DOCX、PPTX、XLSX；MarkItDown `0.1.6` 支持常见文本、PDF、DOCX、PPTX、XLSX 和 XLS。未验证或不支持的格式不进入自动成功状态。

### 4. Extraction Quality Gate

每次解析后自动检查：

- 是否为空或明显过短；
- 输出长度是否与预检抽样文字量严重不符；
- 图片主导文档是否仍需视觉检查；
- 表格结构是否被保留；
- 标题、页码或幻灯片定位是否可用；
- 是否需要切换备用解析器。

图片占比使用抽样页数作为分母，不能把抽样图片页数除以全文页数。若所有已配置解析器都低于质量闸，本次结果不得写入规范化参考文件；保留原件、尝试记录和失败原因。若此前已有通过质量闸且与源文件哈希匹配的参考文件，失败或依赖暂时缺失不能删除它。

每个规范化参考文件必须有 `hxy-parser-reference.v1` 侧车清单，绑定源路径、源文件 SHA-256、参考文件 SHA-256、解析器和质量结果。发现流程只接受哈希仍匹配的参考文件；无清单、源文件已变化或参考文件被修改时，自动重新进入解析路由，不能沿用旧文本。MinerU 每次尝试使用独立工作目录，重试不得复用历史 Markdown 产物。

该质量分只表示“可否进入理解层”，不评价资料真实性、权威性、业务价值或是否适合荷小悦。

### 5. Semantic Understanding

后续强模型不直接扫全部原始目录，而是接收“资料理解包”：

```text
source identity
source class and authority state
business scope and stage
normalized content
page/slide/sheet anchors
parser and quality signals
allowed and blocked use
current business task
```

模型按任务输出少量可用资产，例如来源摘要、与首店的关系、待验证假设、冲突、风险和可行动建议。默认不批量制造 claims，不把一本外部书拆成数千条待审核知识。

### 6. Knowledge Governance

正式知识仍按四个小范围业务包治理：品牌核心、产品边界、首店经营假设、接待与合规。解析成功不触发自动批准；人工只处理核心结论变化、高风险对外口径和机器无法解决的异常，不逐文件确认解析器选择。

`/root/hxy/knowledge/raw/inbox` 当前有 697 个路径。源文件治理后为 674 个内容组；排除重复路径、解析产物和工具文件后，有 509 份唯一有效资料进入解析路由：

```text
hxy_text_compiler  364
markitdown           83
mineru               26
ocr_or_vision        36
```

复杂度分布：

```text
simple  398
medium   49

```

这说明 MinerU 只应优先处理约 5% 的唯一资料，不存在全量重跑的必要。当前文本、MarkItDown 和 MinerU 共 473 份已有执行路径；36 份图片已经得到正确工具路由，但 OCR/视觉执行适配器尚未接入，必须作为 `pending_adapter` 单独统计，不能误报为处理完成，也不需要人工逐张确认。

## 自动化与人工边界

自动完成：

- 去重和排除工具/解析产物；
- 文件结构预检；
- 解析器选择和备用解析；
- 解析质量检查；
- 结构化资料包生成；
- 低风险来源级分类建议。

只在以下情况产生人工任务：

- 所有解析器均失败或格式未支持；
- 核心品牌、产品、融资、法律或合规口径准备晋升；
- 新结论与正式知识冲突；
- 高风险内容准备对外发布；
- 自动分类置信度低且会影响访问权限或权威等级。

## 隐私与边界

- 所有原件和衍生产物留在 HXY 自有目录；
- 不读取或写入 `/root/htops` 业务数据；
- 私有路由报告写入 `data/private/source-routing/`；
- 路由报告和解析结果均 `official_use_allowed=false`；
- 不运行 migration `019`，不写正式知识数据库；
- 不自动批准、发布或修改核心知识。

## 验收标准

1. 所有非重复、非工具、非解析产物的文件都有可解释路由或明确异常；
2. MinerU 只用于结构信号支持的文件或轻量解析失败后的回退；
3. 旧版 DOC/PPT/XLS 不被错误路由到 MinerU；
4. 抽样 PDF 的图片占比使用抽样页数计算；
5. 低质量主解析会自动尝试备用解析器；
6. 所有解析器均失败时，不合格产物不能进入理解层；
7. reference 与当前源文件哈希不匹配时不能进入编译层；
8. 解析失败不能删除此前有效且有证据绑定的 reference；
9. 解析器输出不能成为正式答案或荷小悦权威事实；
10. 不修改任何原始文件，不产生全量 MinerU 任务；
11. 当前路由统计可重复生成且无错误；
12. 相关单元测试、编译检查和差异检查通过。
