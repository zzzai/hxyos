# HXY 超智大脑目标架构

## 设计目标

以终为始，HXY 超智大脑最终要支撑荷小悦从 0 到万店规模。

终局能力不是“问答”，而是：

```text
感知现实 -> 理解业务 -> 发现问题 -> 生成动作 -> 推动执行 -> 追踪结果 -> 修正认知 -> 复制成功
```

## 大厂方案借鉴

### 火山引擎 Data Agent

火山引擎 Data Agent 的核心启发是：数据智能体不能只回答数据问题，而要把自然语言、指标理解、分析执行、洞察生成和行动建议串起来。官方产品表达强调“企业级 AI 数据专家”，并把主动思考、分析和行动作为产品方向。HXY 借鉴其“懂数据、会分析、能行动”的路径。

落到 HXY：
- 自然语言入口不能只走 RAG，要先进入语义前门。
- 数据查询必须经过指标口径、数据覆盖和权限检查。
- 输出不能停在解释，要带行动建议、负责人、验证指标和复盘入口。

### 百度千帆 / 企业 Agent 平台

百度千帆 / DataBuilder 类平台的核心启发是：企业级 AI 应用需要模型服务、Agent 编排、知识增强、安全治理、数据管理、数据加工和数据应用飞轮。HXY 不应绑定单一模型，而应保留模型路由与治理层。

落到 HXY：
- 原始资料、结构化 claim、OSI、指标、动作 SOP 要分层存储。
- 模型只是推理层，项目事实和语义资产必须可迁移。
- 每次回答、执行和复盘都要沉淀为可治理的数据资产。

### 阿里 DataWorks / BI Copilot 类方案

阿里数据智能类方案的核心启发是：数据问答必须建立在数据资产、指标口径、权限、血缘、质量和治理上。HXY 的经营问答必须先有指标语义和数据覆盖判断。

### 腾讯云 Data Agent 类方案

腾讯数据智能体类方案的核心启发是：企业数据智能体应面向业务场景，支持分析、洞察、报告、归因和决策辅助。HXY 应将数据智能体嵌入门店经营、选址、加盟和督导流程。

## 总体架构

```text
用户/任务入口
  - 老板驾驶舱
  - 品牌策划台
  - 店长助手
  - 总部运营台
  - 加盟管理台
  - 数据问答入口

        ↓

语义前门 Semantic Frontdoor
  - 问题类型识别
  - 阶段识别：筹备期/样板店/复制期/规模期
  - 能力路由：知识问答/数据分析/策略生成/动作闭环
  - 可答性检查：证据、数据、权限、口径

        ↓

证据与数据路由 Evidence/Data Router
  - HXY 项目事实
  - 华与华/营销/管理理论
  - 门店经营数据
  - 外部环境数据
  - 竞品与行业数据

        ↓

HXY 轻量本体与语义层
  - 实体
  - 关系
  - claim
  - 版本
  - 冲突
  - 指标
  - recipe

        ↓

数据智能体 + 业务智能体
  - Data Agent
  - Brand Agent
  - Store Model Agent
  - Site Selection Agent
  - Operation Agent
  - Franchise Agent
  - Finance Agent

        ↓

模型路由与推理层
  - 强推理模型
  - 低成本问答模型
  - 多模态模型
  - embedding
  - reranker
  - SQL / 数据分析执行器

        ↓

输出与动作闭环
  - 策略方案
  - 经营诊断
  - 任务清单
  - 话术/SOP
  - 验证指标
  - 复盘与知识更新
```

## 最优解原则

1. **真实数据优先于 PPT 假设**
2. **项目事实优先于外部理论**
3. **外部理论用于提升认知，不直接替代判断**
4. **每个结论必须有证据等级**
5. **每个建议必须有验证指标**
6. **每次执行结果都要回流知识系统**
7. **模型可替换，语义资产不可丢**

## HXY 的“超智大脑”不是单个 Agent

HXY 的最优架构不是把所有问题丢给一个大模型，而是形成五层资产：

1. 项目知识资产：原始资料、会议纪要、PPT、合同、调研、品牌文件。
2. 语义资产：claim、entity、relation、evidence、OSI、指标 contract。
3. 业务资产：品牌全案、终端执行手册、小店模型、样板验证矩阵。
4. 数据资产：订单、会员、复购、技师、私域、财务、选址、竞品、外部环境。
5. 反馈资产：执行任务、回执、经营结果、失败样本、复盘结论。

大模型只负责在这些资产之上做推理、综合和生成，不能替代资产本身。
