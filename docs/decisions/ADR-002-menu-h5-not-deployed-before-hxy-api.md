# ADR-002：菜单 H5 在 HXY API 独立前不作为部署入口

日期：2026-06-26

## 背景

`apps/menu-h5/` 仍保留历史菜单、员工、技师和管理页面。这些页面调用旧的 `/api/v1/wellness/*` 接口，而 HXY 当前阶段的重点是 `0-1 项目推进器` 和品牌知识资产，不是线上点单、门店订单和技师履约。

继续把菜单 H5 作为 nginx 主入口，会产生两个问题：

- 页面能打开，但核心接口不可用。
- 容易把 HXY 重新拉回 htops/wellness 旧运行路径。

## 决策

当前部署入口只保留 HXY 知识 API 和 admin-web：

- `/startup.html`
- `/brain.html`
- `/knowledge.html`
- `/api/*`
- `/health`

菜单 H5 源码暂时保留为待迁移资产，但不通过 nginx 发布为主入口。

## 恢复条件

菜单 H5 只有在以下条件满足后再发布：

1. HXY 自有 API 路由完成，例如 `/api/v1/hxy/*`。
2. 页面不再调用 `/api/v1/wellness/*`。
3. 菜单、订单、门店、员工、技师数据全部写入 HXY 独立数据库。
4. 有菜单 H5 的端到端验证报告。
