# ADR-001：HXY 与 htops 完全分离

## 状态

Accepted

## 日期

2026-06-10

## 背景

荷小悦 HXY 与荷塘悦色是两个独立主体，不是同一公司，也不是同一业务项目。

历史上 HXY 菜单、知识库、Memory Service 和部分 API 被放入 `/root/htops`，导致项目边界、数据边界、服务命名和数据库所有权混乱。

## 决策

创建独立 HXY 项目根：

```text
/root/hxy/
```

`/root/htops` 只保留荷塘悦色业务。

HXY 后续所有菜单、项目知识、Memory Service、API、门店、订单、会员、技师、运营数据都必须进入 `/root/hxy` 和 HXY 独立数据库。

## 后果

正向影响：

- HXY 和荷塘悦色数据完全隔离。
- HXY 菜单系统可以按百店规模独立演进。
- htops 回到荷塘悦色经营内核。
- 未来可以从 HXY 内抽取通用能力，但不共享业务数据。

代价：

- 需要迁移历史 HXY 文件。
- 需要拆出 HXY API。
- 需要新建 HXY 数据库和服务部署。
- htops 中的历史污染面需要在验证后清理。

## 执行原则

```text
复制优先。
验证优先。
备份优先。
不先删除。
```
