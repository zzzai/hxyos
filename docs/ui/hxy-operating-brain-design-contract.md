# HXY Operating Brain Design Contract

## Design Read

荷小悦经营大脑是面向内部团队的经营结果交付台，不是营销落地页，也不是普通资料库。界面应体现低饱和浅色、疗愈、干净、可信任，同时保持企业级工具的效率和清晰。

当前阶段主入口是 `apps/admin-web/index.html`，默认进入 `apps/admin-web/startup.html`。`startup.html` 是 `0-1 项目推进器`，第一屏只服务核爆点定位、清泡调补养口径和品牌资料沉淀。`brain.html` 可以承载通用问答、资料、训练和复核能力，但默认产品感知不能像一个大而全后台。

开店前不把招商、门店日报、客户消费数据作为主入口。这些能力可以保留在折叠后台、API 或后续阶段，但不能抢当前阶段的主任务。

## Visual Direction

- 使用低饱和浅色作为大底，避免刺眼的纯白大面积铺底。
- 背景采用带灰度的莫兰迪色系，例如浅米灰、淡黄、灰粉和浅荷绿。
- 不使用纯白大底，白色只用于承载关键内容的浮层或卡片。
- 整体情绪是 Apple 级克制：安静、准确、少装饰、少噪声。
- 页面核心身份是经营结果交付台，而不是 AI 工具展示台。

## Surface Rules

- 主内容面板使用柔和白色或半透明浅色，不使用重边框。
- 卡片允许使用大圆角，但必须统一半径体系；重复内容卡和操作区可使用 16px，胶囊控件使用 999px。
- 阴影使用弥散投影，颜色要带背景色相，不使用纯黑硬投影。
- 内容区之间优先用留白、浅色背景和柔和阴影分层，不依赖生硬分割线。

## Interaction Rules

- 顶部经营入口和场景切换采用悬浮胶囊操作栏。
- 主按钮使用低饱和深荷绿色，强调稳定和信任。
- 次按钮保持浅底细边，不能抢经营结果卡的层级。
- 上传、导入、纠偏、沉淀答案卡都必须有明确状态反馈。

## Information Hierarchy

主聊天区默认只展示：

- 经营结果
- 可直接使用版本
- 风险边界
- 质量闸口
- 可执行动作

来源、文件路径、chunk、OCR 和原始资料只出现在内部复核折叠区。视觉上要让用户先看到稳定结果，再按需展开复杂依据。

## HXY-Specific Tokens

- 背景：低饱和浅米灰 + 灰粉 + 浅荷绿。
- 主色：深荷绿。
- 强调：少量草本金，不做大面积金色奢华风。
- 状态：成功用荷绿，警示用草本琥珀，错误用低饱和红。

## Anti-Patterns

- 不做 AI 紫蓝渐变。
- 不使用冷硬后台蓝灰风。
- 不把页面做成营销 hero。
- 不把证据、路径、chunk 放在主答案卡上。
- 不把浅色界面做成廉价纯白表单。
